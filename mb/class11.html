  6. Work on planning for your projects and begin coding. Here are some pointers:
    • Discuss roles for each member of your team. Here are a few suggestions:
      • Planner/organizer -- helps coordinate members of the group and maintains focus on specific tasks.
      • Lead programmer -- writes code and hosts the main repository for the group on Github.
      • Lead graphics/designer -- communicates with programmer to determine pixel size of various assets.
      • Support members to handle programming, ideas, research, and graphics components as necessary.
      • Consider two classes: an Asteroid class and a Spaceship class (for example). Two different team members can write these functions separately, then the lead programmer can integrate them into the project.
      • Use Collab Edit for working on the same file at the same time.
    • Break your game down into manageable chunks and figure out how to accomplish those small bits one at a time. This is an extremely important process in coding. For example, when I wrote Tetris, I approached the game (roughly) in the following sequence of steps:
      1. Set up variables, constants, and basic functions and objects
      2. Get a square block to appear on screen
      3. Get that block to move down the screen
      4. Add keyboard input
      5. Handle collisions for the active block to prevent it from going off the edges and to make it inactive when it reaches the bottom
      6. Add more block types and write rotation functions for each block type
      7. Add scores, images, and a second canvas to hold the next block
